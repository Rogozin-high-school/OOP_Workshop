בנאי המחלקה

כשרצינו ליצור אובייקט חדש של המחלקה Car, זה לקח לנו 4 שורות: 
Car c = new Car();
c.Model = ...
c.Company = ...
c.ProductionYear = ...
חייבת להיות דרך נוחה יותר...
כשאחנו כותבים new Car עם סוגריים ריקות, אנחנו בעצם קוראים לפעולה, והפעולה הזו אחראית "ליצור" את המחלקה.
הפעולה הזו נקראת בנאי של המחלקה.
אם לא הגדרנו למחלקה בנאי משלנו, לא יקרה שום דבר מיוחד כאשר ניצור אובייקט חדש בעזרת new Car().
אבל מה אם כן נגדיר בנאי משלנו?

הגדרת הבנאי

הבנאי של המחלקה (Constructor) מוגדר בצורה כזו: 
class Student
{
	...
	
	public Student( < parameters > )
	{
	
	}
}
כאשר במקום שומר המקום parameters, הבנאי יכול לקבל פרמטרים, כמו כל פעולה רגילה.
אבל, הבנאי לא יכול להחזיר ערך, ובגלל זה גם לפעולת הבנאי אין שום ערך החזרה - אפילו לא void.
שם הבנאי חייב להיות זהה לשם המחלקה.
למחלקה יכולים להיות כמה בנאים.

לדוגמא, נכתוב בנאי למחלקה Student שיקבל שני פרמטרים - שם התלמיד והציון שלו. 
class Student
{
	private string Name;
	private int Grade;
	
	... getters, setters ...
	
	public Student(string name, int grade)
	{
		SetGrade(grade);
		SetName(name);
	}
}
השימוש בפעולות הגישה בתוך הבנאי הוא רק כדי שלא נצטרך להעתיק את הקוד של אימות הקלט פעמיים. בתוך הבנאי היה ניתן להשים את הערכים ישר אל תוך השדות, להדפיס הודעות, ליצור משתנים, ולעשות כל דבר שאפשר בפעולה רגילה.

הגדרת בנאי למחלקה Car

הגדירו בנאי למחלקה Car שיקבל שלושה פרמטרים, וישים את הערכים שלהם בשדות המחלקה.
* אין צורך באימות קלט או בשימוש בפעולות גישה.

כשתסיימו, כנראה שתשימו לב לשגיאה הבאה: 
'Workshop.Car' does not contain a constructor that takes 0 arguments
למה זה קורה?
כאשר אנחנו לא מגדירים בנאי משלנו, C# מגדיר אוטומטית עבורינו בנאי שלא מקבל פרמטרים, ולא עושה כלום.
ברגע שהגדרנו בנאי משלנו שכן מקבל פרמטרים, הבנאי שלא מקבל פרמטרים נעלם, כי כבר יש בנאי אחר.
לכן זה לא אפשרי לקרוא לnew Car ללא פרמטרים, חייבים לספק את הפרמטרים שהבנאי שבנינו דורש.
איך נפתור את זה?

נשתמש בעובדה שלמחלקה מותר להכיל יותר מבנאי אחד!
נגדיר בנאי נוסף, שלא יקבל פרמטרים, ולא יעשה כלום.
נסו להגדיר במחלקה Car בנאי שלא יקבל פרמטרים, ויהיה ריק - שלא יעשה דבר. אם הבנאי לא עושה דבר, ערכי השדות של המחלקה יהיו ערכי ברירת המחדל של הטיפוסים.
כאשר סיימתם, לחצו "המשך"
