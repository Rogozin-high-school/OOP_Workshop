תכנות מונחה עצמים

בואו ננסה לייצג תלמיד בעזרת משתנה.
נניח, לכל תלמיד יש  שם  ו ציון.
הדרך הראשונה שאפשר לנסות היא לייצג את התלמיד בעזרת שני משתנים: 
int stuGrade = 93;
string stuName = "Dani";
נראה בסדר, לא? אבל מה קורה כשיש לנו פעולות שאמורות לעבוד עם הטיפוס? למשל, נכתוב פעולה בשם RegisterToSchool. הפעולה צריכה לקבל את התלמיד שעליה לרשום. אבל התלמיד אצלינו מיוצג על ידי שני משתנים, ולכן נצטרך לשלוח לפעולה שני פרמטרים..
לא ממש נוח...
ואם לתלמיד היו מערך ציונים, שם, כיתה, כתובת, אימייל וטלפון? היינו צריכים לשלוח 6 פרמטרים, וכל הגדרה של תלמיד הייתה 6 משתנים שונים..
בנוסף, זה לא ממש קריא ומסרבל את הקוד שלנו.
בגלל זה, אנשים חכמים המציאו את התכנות מונחה העצמים, ובראשו המחלקה

מהי מחלקה?

דמיינו מחלקה כמו תבנית, המתארת סוג חדש של משתנה. למשל - מחלקת תלמיד. המחלקה לא מתארת תלמיד ספציפי שציונו X ושמו Y.
המחלקה מתארת אילו שדות ומשתנים יש לאותו "תלמיד"

נתחיל עם דוגמה: 
class Student
{
	public int Grade;
	public string Name;
}
זוהי הדוגמה הפשוטה ביותר שאפשר לתת למחלקה - בעצם, ברגע שיצרנו מחלקה, אפשר לחשוב על זה כך, כאילו יצרנו "טיפוס" חדש של משתנה.
עוד לא יצרנו שום משתנה מהסוג הזה, רק הגדרנו שיש סוג משתנה חדש , קוראים לו Student , והוא מכיל שני "ערכים" בתוכו - Grade וName
שימו לב לתחביר הפשוט - בדומה להגדרת פעולה, תוכן המחלקה נכתב בתוך סוגריים מסולסלות.
בואו נמשיך ונראה איך נשתמש בטיפוס הנתונים החדש שיצרנו.

יצירת משתנים מטיפוס Student

עכשיו, לאחר שהגדרנו את הטיפוס שלנו, בואו נכתוב פונקציית Main, ובה ננסה להשתמש בטיפוס הנתונים החדש שלנו. 
public static void Main(string[] args)
{
	Student stu;
}
ככה מגדירים משתנה שהוגדר באמצעות מחלקה. מוכר? בדיוק כמו הגדרת משתנה רגיל!

לפני שנמשיך, בואו ננסה להגדיר מחלקה בעצמינו - לחצו "המשך" כדי להגיע לתרגול

נסו זאת בעצמכם - הגדרת מחלקה

הגדירו מחלקה בשם Car. למחלקה יהיו שלושה שדות (ערכים) - שם דגם, שם חברה ושנת ייצור.
Model - string
Company - string
ProductionYear - int
הקפידו על שמות השדות שיתאימו לשמותם בתרגיל, ועל טיפוסי הנתונים.
רשמו את הקוד שלכם בעורך הקוד שמשמאל. כשאתם מסיימים את התרגיל, לחצו Run וראו אם הצלחתם!
